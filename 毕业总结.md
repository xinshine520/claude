# 🎓 AI 编程实战营第一期：从“AI 使用者”到“AI 开发者”的蜕变总结

## 📝 个人背景与现状

目前我是一名深耕于软件开发领域的工程师，主要在 **Windows (WSL/MSYS2)** 环境下进行开发，技术栈涵盖 **Go、Java、Python** 及 **Node.js**。在参加实战营之前，虽然我已经开始接触 AI 工具，但主要停留在简单的对话层面，对于如何将 AI 深度嵌入到复杂的开发工作流中仍有困惑。

## 🚀 为什么选择报名学习？

1. **系统化进阶**：希望超越简单的 Prompt Engineering，掌握 **AI Agent** 和 **MCP（Model Context Protocol）** 等前沿技术。
2. **解决实战痛点**：在处理大型遗留代码库和复杂数据库查询时，急需更智能的方案来提升效率。
3. **构建全流程闭环**：渴望掌握从需求分析到代码实现、再到部署的全链路 AI 赋能手段。

## 💡 课程实战收获：四大核心模块

通过本学期的四个关键实战项目，我的技术视野得到了彻底重构：

* **实战一：智能数据库查询与全流程实践**
我深入学习了如何构建 **NL2SQL** 服务。这让我能够利用 Python 和 LLM 将自然语言精准转化为 PostgreSQL 查询语句，大幅提升了数据分析的效率。
* **实战二：大型代码库理解 / MCP 服务器**
这是最令我惊喜的部分。通过学习 MCP 服务器的构建，我学会了如何让 AI 更好地“读懂”我的项目结构。现在，即便是面对复杂的工程，我也能通过自定义上下文快速定位问题。
* **实战三：构建你的专属 Agent**
在这个环节，我基于 **ReAct 框架** 搭建了专属 Agent。通过对 **OpenClaw** 等工具的调试，我理解了 AI 代理如何自主进行工具调用和任务编排，真正实现了从“问答”到“执行”的跨越。
* **实战四：端到端 AI 赋能的软件开发全流程实践**
这个综合项目将一切串联了起来。我尝试将 **Claude Code**、**Cursor** 与 Git Hooks 结合，在 Go 和 Java 项目中实现了从代码生成到规范检查的半自动化流转，开发效率提升了 50% 以上。

## ✅ 解决的工作/项目关键问题

* **开发环境调优**：在助教的帮助下，我完美解决了 WSL 资源分配及 pnpm 代理配置等顽疾，打造了极简的终端开发体验。
* **生产力工具落地**：我成功开发了一套针对公司内部数据库的智能查询工具，得到了团队内部的认可。
