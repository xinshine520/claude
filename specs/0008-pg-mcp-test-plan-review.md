## 1. 总体评价

### 1.1 做得好的方面

- **分层清晰 + marker 策略合理**：能把秒级反馈的 unit 与依赖 Docker 的 integration/e2e 解耦
- **安全测试作为主线**：validator 用例密度高，覆盖 DML/DDL、多语句、CTE 夹带、危险函数、注释绕过、EXPLAIN ANALYZE 等关键向量；将"校验失败不回传 SQL（PL8/E8）"显式化，是非常重要的安全行为约束
- **纵深防御覆盖面完整**：AST 白名单 + 函数黑名单 + PG 只读事务 + 超时 + 脱敏 + 熔断 + 并发限制，§7 做了"防御层验证矩阵"汇总
- **集成测试贴近真实风险**：不只覆盖"能查到数据"，还覆盖 `search_path`、字段截断、payload 裁剪、错误脱敏、prepare 类型信息等真实 PG 行为
- **CI 分层设计合理**：lint / typecheck / unit / integration / security 分离，并将覆盖率门槛落到命令行参数

### 1.2 用例统计

约 235 个测试用例，其中安全相关 84 个（与其他层有重叠），Validator 单项 65+ 用例。

### F5: Validator 边界/绕过测试可扩展

| 属性     | 值                                                                 |
| -------- | ------------------------------------------------------------------ |
| 位置     | §4.5 test_validator.py                                             |
| 严重度   | **Medium**                                                         |
| 描述     | 当前 validator 用例密度高且覆盖主要攻击向量，但作为"安全核心"组件可进一步系统化。缺少以下常见绕过/边界场景。 |

|          | - V122: 多行注释穿插关键字 → `SELECT 1 /* DROP TABLE x */` → 通过 |

### F7: 集成测试隔离机制缺少具体 fixture 说明

| 属性     | 值                                                                 |
| -------- | ------------------------------------------------------------------ |
| 位置     | §9.2 测试隔离                                                      |
| 严重度   | **Medium**                                                         |
| 描述     | 写了"每个测试使用独立事务 + ROLLBACK"，但未说明由哪个 fixture 保证、异常时是否也会 rollback、连接是否必定 release。实现时可能遗漏导致脏数据或连接泄漏。 |
| 建议     | 补充具体 fixture 设计：                                            |

```python
@pytest.fixture
async def pg_txn(pg_conn):
    """每个测试在独立事务中运行，测试结束后自动 ROLLBACK"""
    txn = pg_conn.transaction()
    await txn.start()
    yield pg_conn
    await txn.rollback()
```

### F8: Docker 端口与 CI 服务端口不一致

| 属性     | 值                                                                 |
| -------- | ------------------------------------------------------------------ |
| 位置     | §3.2 docker-compose.yml vs §3.4 conftest.py vs §11.1 CI           |
| 严重度   | **Low**                                                            |
| 描述     | docker-compose.yml 映射到 `15432:5432`，conftest.py DSN 使用 `localhost:15432`，但 CI 的 GitHub Actions services 通常直接暴露 `5432`。两者端口不一致会导致 CI 集成测试连接失败。 |
| 建议     | conftest.py 中使用环境变量获取端口：`os.getenv("PG_MCP_TEST_PORT", "15432")`，CI 中设置 `PG_MCP_TEST_PORT=5432`。 |
